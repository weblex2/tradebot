# Session 2026-03-21c

## Änderungen

### Feature: Technische Analyse für den Analyser

**Neuer Service: `app/Services/TechnicalAnalysisService.php`**
- Berechnet alle Indikatoren manuell (keine externe Library)
- `buildPromptBlock(symbol, dailyCandles, hourlyCandles)` → kompakter Text-Block für Claude
- Indikatoren:
  - **RSI (14, stündlich)** — Wilder's Smoothing, Label: oversold/bearish/neutral/bullish/overbought
  - **MACD (12/26/9, stündlich)** — EMA-basiert, Histogramm-Label: bullish/bearish cross
  - **SMA 20 + SMA 50 (täglich)** — Golden/Death Cross Erkennung, % Abstand vom aktuellen Preis
  - **Bollinger Bands (20, täglich)** — Position: above upper / near upper / mid / near lower / below lower
  - **Support/Resistance (14 Tage täglich)** — Min Low / Max High der letzten 14 Tage

**`CoinbaseService` erweitert:**
- `getCandles(symbol, granularity, start, end)` — neue generische Candle-Methode
- `getHourlyCandles(symbol, hours=60)` — 60 Stunden Hourly Candles für RSI/MACD
- `getVolumeData()` auf 55 Tage erweitert (war 8) für SMA50-Berechnung
- `daily_candles` im Return-Wert — wird in TA wiederverwendet (kein zweiter API-Call)
- 7-Tage-Durchschnitt fix: `array_slice($volumes, 1, 7)` statt aller Candles

**`ClaudeAnalysisService::runAnalysis()` erweitert:**
- Neuer Parameter `array $technicalData = []`
- Neuer Prompt-Abschnitt `## Technical Analysis` mit allen Indikatoren
- System-Prompt-Regeln für TA-Interpretation ergänzt (RSI overbought/oversold, MACD, BB, SMA Cross)

**`TradeAnalyzeCommand` erweitert:**
- `TechnicalAnalysisService` per Constructor Injection
- Hourly Candles pro Asset nach Volume-Fetch (1 API-Call pro Asset, 150ms sleep)
- Daily Candles werden aus `volumeData['daily_candles']` wiederverwendet
- `signals_summary` in DB enthält jetzt auch `technical` Key

## Beispiel-Output (BTC)
```
[BTC Technical]
RSI14(1h)=44.3 neutral
MACD(1h): line=-0.2995 signal=50.7051 hist=-51.0047 bearish
SMA20=70,350.26 SMA50=69,565.93 (price -0.1% vs SMA20, golden cross)
BB: upper=74,626.93 mid=70,350.26 lower=66,073.58 width=12.2% mid
S/R(14d): support=65,618.51 resistance=76,022.60 (+7.1% from support, -7.5% from resistance)
```

## Architektur-Entscheidung
- Keine DB-Speicherung der Indikatoren — Preisdaten veralten sofort
- Nur zur Nachvollziehbarkeit im `analyses.signals_summary` JSON gespeichert
- Kein neuer DB-Table nötig
